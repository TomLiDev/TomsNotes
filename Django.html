Django

pip3 - Using 3 with the pip command installs the specified package for Python version 3, without this it would install Python version 2.

Django is a major framework which has lots of pre-built out of the box components and cool shit.

pip3 install django - terminal command to install newest version of Django

pip3 install 'django<4' - terminal command to install Django 3.2 which is the long term support version of Django.

django-admin startproject folder_name . - This command from the terminal will create a project in the current directory. This will automatically create certain files in the project folder, such as __intit__py, settings.py etc.

python3 manage.py runserver - Terminal command to actually run the application. May require some config on the settings.py to be changed.

views.py - file that is provided with Django. Function can be written which will render HTML and other code.

urls.py - file that when connected with a view (above) allows the python function (which may render html etc) written in views to be called when a specific url is entered.

render function - built in function of Django (and other frameworks) takes 2 arguments, first is a HTTP request, second is a template name. It returns a HTTP response which renders the specific template.
The below line is an example of a return line from a python function which is rendering a specific html file found at the pathway specified. return render(request, 'todo/todo_list.html')

python3 manage.py makemigrations - Terminal command to convert python code into SQL code. with --dry-run added at the end, this will not actually make migrations but will highlight any changes that would be made by running the command. makemigrations essentially sets up the database and creates an admin user to manage it.

python3 manage.py showmigrations - Terminal command to show migrations which need to be made, outstanding.

python3 manage.py migrate - Performs necessary migrations

python3 manage.py createsuperuser - Terminal command to create admin/database user. Once this has been done, the Django admin panel for the site can be accessed by simply adding /admin onto the website URL.

Django Data Stuff

models.py - Automatically created file with Django, this is where data models are created and managed. A class at this level is very similar conceptually to an individual sheet in an excel doc, with the entire excel being the database, therefore a data model is comparable to a sheet. This file automatically imports the model class from Django.

from django.db import models
# Create your models here.
class Item(models.Model):
The above is a basic example of the Django models functionality. In this case the manually created Item class in inheriting the base model class from the imported Django model class.

class Item(models.Model):
name = models.CharField(max_length=50, null=False, blank=False)
done = models.BooleanField(null=False, blank=False, default=False)
The above is an extended example of using inheritance to create a data model from the Django class model. In this example the custom, manually created attributes of name and done (intended as a status against to do items) have been added to the Items class. These have been customised using standard values from Django, such as max characters, and settings which prevent an empty value being entered.
After this item call was created, python3 manage.py makemigrations was run to actually create the migration from the model provided. This also creates a new .py file.
Then python3 manage.py migrate was used to create the database table itself.

{{ item }} - When used in a html template with Django, this double curly brackets syntax allows things from a dictionary to be rendered in the page. This is called a template variable.

{% for item in items %} - Using curly brackets and percentage signs allows creation of for loops directly within the html template
{% endfor %}

{% if %} {% else %} - Same syntax as above can be used for creating if/else loops.
